JavaScript 基础教程

JavaScript（简称 JS）是 Web 的编程语言，让网页具备交互能力：响应点击、验证表单、动态更新内容、与服务器通信等。

1. 什么是 JavaScript？

继续用盖房子的比喻：HTML 是框架，CSS 是装修，JavaScript 是电路和智能系统——灯能开关、门铃能响、温度能自动调节。

JavaScript 能做什么？

  • 响应用户操作（点击按钮、输入文字、滚动页面）
  • 动态修改页面内容和样式
  • 验证表单数据
  • 创建动画和特效
  • 与服务器通信（发送/接收数据）
  • 存储数据到浏览器（localStorage）

JavaScript 在哪里运行？

  • 浏览器中：前端开发的主要场景（Chrome、Firefox、Edge 都内置 JS 引擎）
  • Node.js：在服务器端运行 JS（后端开发）
💡 调试工具 按 F12 打开浏览器开发者工具，切换到「Console（控制台）」标签页，可以直接输入 JS 代码并查看输出。学习 JS 时这是你的好帮手。

2. 如何引入 JavaScript

方式一：外部脚本（推荐）

HTML
<!-- 放在 </body> 前，确保 HTML 先加载 -->
<script src="js/app.js"></script>
</body>

方式二：内部脚本

HTML
<script>
  console.log('Hello, JavaScript!');
</script>
⚠️ 注意 使用 src 属性的 <script> 标签内不能再写代码。脚本通常放在 </body> 前，避免阻塞页面渲染。

7. 函数

函数（Function）是 JavaScript 中组织代码的基本单元。你可以把一段逻辑封装起来，给它起个名字，需要时反复调用。几乎所有 JS 程序都离不开函数。

为什么需要函数？

  • 复用：同一段逻辑写一次，多处调用（如「格式化日期」）
  • 抽象：调用者只需知道「做什么」，不必关心「怎么做」
  • 维护：修改逻辑只改一处，不用到处找重复代码
  • 组织：把大程序拆成小块，更易阅读
JavaScript
// 没有函数：重复代码
console.log('欢迎，张三！');
console.log('欢迎，李四！');

// 有函数：封装后复用
function welcome(name) {
  console.log(`欢迎，${name}！`);
}
welcome('张三');
welcome('李四');

参数的传递与 return

函数通过参数（parameter）接收外部数据，通过 return 把结果返回给调用者。

概念说明示例
形参 parameter定义函数时括号里的变量名function add(a, b) 中的 a、b
实参 argument调用函数时传入的实际值add(3, 5) 中的 3、5
返回值return 后面的表达式结果没有 return 则返回 undefined
JavaScript
function add(a, b) {
  const result = a + b;
  return result;       // 把结果返回给调用者
  console.log('不会执行'); // return 之后的代码不会运行
}

const sum = add(3, 5);  // sum = 8

function logMessage(msg) {
  console.log(msg);
  // 没有 return，默认返回 undefined
}

function earlyExit(n) {
  if (n < 0) {
    return '无效';  // 提前返回，函数立即结束
  }
  return n * 2;
}

三种定义函数的方式

方式语法特点
函数声明 function fn() { } 会被提升（hoisting），可在声明前调用
函数表达式 const fn = function() { } 不会提升，必须先定义再调用
箭头函数 const fn = () => { } 语法简洁，没有自己的 this（见下文）
JavaScript
// 1. 函数声明 — 可以先调用后定义
sayHello('小明');  // ✅ 不报错（提升）

function sayHello(name) {
  return `你好，${name}！`;
}

// 2. 函数表达式 — 必须先定义
const add = function(a, b) {
  return a + b;
};

// 3. 箭头函数 — 现代写法，最常用
const multiply = (a, b) => a * b;           // 单行可省略 {} 和 return
const square = x => x * x;                    // 单参数可省略括号
const sayHi = () => console.log('Hi!');       // 无参数必须写 ()

const calc = (a, b) => {                      // 多行必须写 {} 和 return
  const result = a + b;
  return result;
};

箭头函数 vs 普通函数

初学阶段最需要记住的区别：

  • 箭头函数没有自己的 this，会继承外层作用域的 this（写 DOM 事件时很有用）
  • 箭头函数没有 arguments 对象，要用剩余参数 ...args 代替
  • 箭头函数不能作为构造函数（不能 new）
JavaScript
// 对象方法：普通函数中 this 指向调用者
const user = {
  name: '张三',
  greet: function() {
    console.log(this.name);  // '张三'
  },
};

// 箭头函数做方法时 this 不指向 user（容易踩坑）
const badUser = {
  name: '李四',
  greet: () => {
    console.log(this.name);  // undefined（this 指向外层，不是 badUser）
  },
};

默认参数、剩余参数与展开运算符

JavaScript
// 默认参数：调用时未传值则使用默认值
function createUser(name, role = '用户', active = true) {
  return { name, role, active };
}
createUser('张三');                    // role='用户', active=true
createUser('李四', '管理员');           // active 仍为 true
createUser('王五', '编辑', false);

// 剩余参数 ...rest：把多余的实参收集成数组
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
sum(1, 2, 3, 4);  // 10

function introduce(name, ...hobbies) {
  console.log(`${name} 的爱好：`, hobbies);
}
introduce('小明', '阅读', '游泳', '编程');
// 小明 的爱好： ['阅读', '游泳', '编程']

// 展开运算符 ...spread：把数组「展开」成独立参数
const nums = [1, 5, 3];
Math.max(...nums);  // 等同于 Math.max(1, 5, 3) → 5

回调函数与高阶函数

回调函数（callback）是作为参数传给另一个函数的函数，在特定时机被「回调」执行。这是 JS 异步编程和 DOM 事件的基础。

JavaScript
// 回调：把函数当参数传递
function doSomething(callback) {
  console.log('开始工作...');
  callback();  // 执行传入的函数
  console.log('工作完成');
}

doSomething(function() {
  console.log('中间步骤');
});

// 数组方法大量使用回调
[1, 2, 3].map(n => n * 2);           // [2, 4, 6]
[1, 2, 3].filter(n => n > 1);       // [2, 3]
[1, 2, 3].forEach(n => console.log(n));

// 高阶函数：接收或返回函数的函数
function repeat(times, fn) {
  for (let i = 0; i < times; i++) {
    fn(i);
  }
}
repeat(3, (i) => console.log(`第 ${i} 次`));

作用域与闭包入门

作用域决定变量在哪些地方可以被访问：

  • 全局作用域：函数外声明，整个脚本可访问
  • 函数作用域：函数内声明，只在函数内有效
  • 块级作用域：let / const 在 { } 内有效；var 没有块级作用域（不推荐用 var）
JavaScript
const globalVar = '全局';

function outer() {
  const outerVar = '外层';

  function inner() {
    const innerVar = '内层';
    console.log(globalVar);  // ✅ 可以访问全局
    console.log(outerVar);   // ✅ 可以访问外层
    console.log(innerVar);   // ✅ 可以访问自身
  }

  inner();
  // console.log(innerVar);  // ❌ 无法访问 inner 内部的变量
}

outer();

闭包（closure）：内部函数「记住」了外部函数的变量，即使外部函数已经执行完毕。

JavaScript
function createCounter() {
  let count = 0;  // 被内部函数「闭包」保存

  return {
    increment() { return ++count; },
    decrement() { return --count; },
    getCount() { return count; },
  };
}

const counter = createCounter();
counter.increment();  // 1
counter.increment();  // 2
counter.getCount();   // 2
// count 变量外部无法直接访问，只能通过返回的方法操作
💡 函数编写建议
  • 函数名用动词开头：getUser、calculateTotal、renderList
  • 一个函数只做一件事，保持简短
  • 优先用箭头函数写回调，对象方法用普通函数或 ES6 简写 greet() { }
  • 参数不宜过多，超过 3 个可考虑传对象

9. DOM 操作

这一章是前端交互的核心：学会用 JavaScript 操作页面上的 HTML 元素。而要做到这一点，必须先理解 DOM 是什么。

先理解：DOM 到底是什么？

DOM（Document Object Model，文档对象模型）是浏览器读取 HTML 文件后，在内存里构建的一棵树形结构。树上的每一个节点，都对应 HTML 里的某个部分。

可以这么理解三者关系：

  • HTML 文件：磁盘上的文本，是「设计图纸」
  • DOM 树：浏览器把 HTML 解析后在内存里的「活的对象结构」
  • 你看到的网页：浏览器根据 DOM 树 + CSS 渲染出来的最终画面
🔑 关键理解 JavaScript 不能直接修改 .html 源文件，但可以修改内存中的 DOM 树。DOM 一变，页面显示就会跟着变——这就是网页能「动起来」的根本原因。

HTML、CSS、JavaScript 与 DOM 的分工

技术职责与 DOM 的关系
HTML 定义页面结构（有什么元素） 提供 DOM 树的初始内容
CSS 定义页面外观（长什么样） 不修改 DOM 结构，只影响渲染样式
JavaScript 定义页面行为（怎么交互） 通过 DOM API 增删改查节点，动态改变页面

打个比方：HTML 是房子的框架，CSS 是装修，JavaScript 是智能家居系统——而 DOM 就是那套可以被 JS 控制的「房屋结构模型」，JS 通过它开关灯、开门窗、添家具。

一个最小例子：JS 如何改变页面

HTML 里写了：

HTML
<p id="msg">原始文字</p>

JavaScript 通过 DOM 找到这个节点并修改：

JavaScript
const msg = document.querySelector('#msg');  // 在 DOM 树中找到节点
msg.textContent = '文字已被 JS 修改了';            // 修改节点内容

用户看到的文字变了，但磁盘上的 .html 文件并没有变——变的是浏览器内存里的 DOM 树。

DOM 树详解：从 HTML 到树

下面是一份完整的 HTML 页面。浏览器加载它之后，会把每个标签、每段文字都变成树上的节点。

HTML
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="utf-8">
  <title>我的网页</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header id="header">
    <h1>欢迎访问</h1>
    <nav>
      <a href="/">首页</a>
      <a href="/about">关于</a>
    </nav>
  </header>
  <main class="content">
    <p>这是一段正文。</p>
    <ul id="list">
      <li>项目 A</li>
      <li>项目 B</li>
    </ul>
    <button id="btn">点击我</button>
  </main>
  <footer>
    <p>© 2026 我的网站</p>
  </footer>
  <script src="app.js"></script>
</body>
</html>

浏览器解析后，生成的 DOM 树如下（蓝色 = 元素节点，绿色 = 文本节点，黄色 = 属性）：

Document ← 整个文档的根，JS 通过 document 对象访问 └── html [lang="zh-CN"] ├── head │ ├── meta [charset="utf-8"] ← 自闭合标签也是元素节点 │ ├── title │ │ └── #text: "我的网页" ← 标签内的文字是文本节点 │ └── link [rel="stylesheet"] [href="style.css"] │ └── body ├── header [id="header"] │ ├── h1 │ │ └── #text: "欢迎访问" │ └── nav │ ├── a [href="/"] │ │ └── #text: "首页" │ └── a [href="/about"] │ └── #text: "关于" │ ├── main [class="content"] │ ├── p │ │ └── #text: "这是一段正文。" │ ├── ul [id="list"] │ │ ├── li │ │ │ └── #text: "项目 A" │ │ └── li │ │ └── #text: "项目 B" │ └── button [id="btn"] │ └── #text: "点击我" │ ├── footer │ └── p │ └── #text: "© 2026 我的网站" │ └── script [src="app.js"]

节点有哪些类型？

节点类型是什么举例JS 中常见吗
Document 整份文档的根节点 document ✅ 入口，必用
Element
元素节点		 HTML 标签 <div>、<p>、<button> ✅ 最常操作
Text
文本节点		 标签内的文字 "欢迎访问"、"点击我" ✅ 通过 textContent 操作
Attribute
属性		 标签上的特性 id="btn"、class="content" ✅ 通过 get/setAttribute
Comment
注释节点		 HTML 注释 <!-- 注释 --> 较少直接操作
💡 注意区分 我们日常说「操作 DOM」时，90% 以上是在操作 Element 元素节点。文本节点和属性是元素的附属部分，通过元素的 API 间接操作。

树中的关系术语

DOM 是一棵树，节点之间有明确的父子、兄弟关系。这些术语在写 JS 时会经常用到：

父节点 / 子节点

body 是 header 的父节点，header 是 body 的子节点。直接包含的关系叫「父子」。

兄弟节点

header、main、footer 同属于 body，它们互为兄弟节点（同级）。

祖先 / 后代

body 是 button#btn 的祖先，button 是 body 的后代。不限层级，隔代也算。

根节点

document 是整棵树的根。所有节点最终都能追溯到 document。

以上面那棵树为例，看 button#btn 这个节点：

  • 父节点：main.content
  • 子节点：文本节点 "点击我"
  • 兄弟节点：p、ul#list（同在 main 下）
  • 祖先节点：main → body → html → document

DOM 操作本质上在做什么？

理解了树结构之后，后面所有 DOM API 都可以归结为对树的四种操作：

操作含义典型 API类比
查 在树中找到目标节点 querySelector、getElementById 在树上找到某根树枝
改 修改节点内容、属性或样式 textContent、classList、setAttribute 改树枝上的标签和内容
增 创建新节点并挂到树上 createElement、appendChild 长出一根新树枝
删 从树上移除节点 remove、removeChild 砍掉一根树枝

下面几节就分别讲这四种操作的具体用法。

📝 动手验证：在 DevTools 里看 DOM 树
  1. 按 F12 打开开发者工具
  2. 切到 Elements（元素） 面板
  3. 左侧就是实时的 DOM 树，点击节点可以展开/折叠
  4. 在 Console 里输入 document.querySelector('#btn')，可以看到对应的 DOM 对象
  5. 用 JS 修改 DOM 后，Elements 面板会立刻同步更新

选择元素（查）

操作 DOM 的第一步是「在树中找到目标节点」。现代开发推荐优先使用 querySelector 系列。

方法返回说明
getElementById('id') 单个元素或 null 按 id 查找，id 在页面中应唯一
querySelector('.class') 第一个匹配或 null 支持任意 CSS 选择器，最常用
querySelectorAll('li') NodeList（类数组） 匹配所有符合的元素
getElementsByClassName('c') HTMLCollection（动态） 按 class 查找，较少用
getElementsByTagName('p') HTMLCollection 按标签名查找
JavaScript
// 按 id — 最快，但只能用于有 id 的元素
const header = document.getElementById('header');

// querySelector — 推荐，支持所有 CSS 选择器
const btn = document.querySelector('.btn');              // class
const firstItem = document.querySelector('ul li');     // 后代
const activeLink = document.querySelector('nav a.active');
const submitBtn = document.querySelector('button[type="submit"]');

// querySelectorAll — 选中多个，返回 NodeList
const allItems = document.querySelectorAll('.list-item');
allItems.forEach(item => {
  item.classList.add('processed');
});

// 在已选中的元素内部继续查找（缩小范围）
const nav = document.querySelector('nav');
const navLinks = nav.querySelectorAll('a');  // 只在 nav 内部找
⚠️ 注意 querySelector 找不到元素时返回 null。对 null 调用方法会报错，使用前先判断： if (btn) { btn.addEventListener(...) }

修改内容和属性（改）

textContent vs innerHTML vs innerText

属性行为使用场景
textContent 获取/设置纯文本，会解析所有子节点的文字 推荐，安全，不受 HTML 标签影响
innerHTML 获取/设置 HTML 字符串，会解析标签 需要插入 HTML 结构时用，注意 XSS 安全风险
innerText 只返回「可见」文字，受 CSS 影响 需要模拟用户看到的文字时
JavaScript
const box = document.querySelector('#box');

// 纯文本（安全）
box.textContent = 'Hello World';

// 插入 HTML（用户输入的内容不要直接 innerHTML！）
box.innerHTML = '<strong>加粗</strong> 文字';

// 标准属性
const img = document.querySelector('img');
img.src = 'photo.jpg';
img.alt = '描述';

const link = document.querySelector('a');
link.href = 'https://example.com';

// 通用属性读写
link.setAttribute('target', '_blank');
link.getAttribute('href');
link.hasAttribute('disabled');
link.removeAttribute('title');

// data-* 自定义属性（HTML5）
// HTML: <div data-user-id="42" data-role="admin">
const el = document.querySelector('[data-user-id]');
el.dataset.userId;   // '42'（自动转为 camelCase）
el.dataset.role;     // 'admin'
el.dataset.status = 'online';  // 设置 data-status="online"

修改样式

有两种方式：直接改 style，或操作 class（推荐）。

JavaScript
const title = document.querySelector('h1');

// 方式一：内联 style（camelCase 写法）
title.style.color = 'red';
title.style.fontSize = '24px';       // 对应 CSS font-size
title.style.backgroundColor = '#f0f0f0';
title.style.display = 'none';         // 隐藏元素

// 方式二：classList（推荐 — 样式写在 CSS 里，JS 只切换类名）
title.classList.add('active');       // 添加类
title.classList.remove('hidden');    // 移除类
title.classList.toggle('highlight'); // 有则删、无则加
title.classList.contains('active');  // true / false
title.classList.replace('old', 'new'); // 替换类名

// 读取计算后的样式（只读）
const styles = getComputedStyle(title);
styles.fontSize;  // 如 '32px'

创建、插入与删除元素（增 / 删）

动态内容（待办列表、评论、弹窗等）都需要用 JS 创建新元素并插入 DOM。

JavaScript
// 1. 创建元素
const li = document.createElement('li');
li.textContent = '新项目';
li.classList.add('list-item');

// 2. 设置属性和内容
li.setAttribute('data-id', '123');

// 3. 插入到 DOM（选一种）
const list = document.querySelector('ul');
list.appendChild(li);              // 末尾追加（传统）
list.prepend(li);                  // 开头插入
list.before(li);                   // 插到 list 前面（同级）
list.after(li);                    // 插到 list 后面（同级）
list.insertBefore(li, refNode);    // 插到 refNode 之前

// 现代写法：insertAdjacentElement / insertAdjacentHTML
list.insertAdjacentHTML('beforeend', '<li>快速插入</li>');
// 位置：beforebegin | afterbegin | beforeend | afterend

// 4. 删除元素
li.remove();                       // 现代写法，元素自己删除自己
list.removeChild(li);              // 传统：父元素删子元素
list.innerHTML = '';               // 清空所有子元素（简单但会移除事件监听）

批量创建：DocumentFragment

循环插入大量元素时，先用 DocumentFragment 在内存中组装，再一次性插入 DOM，性能更好。

JavaScript
const fragment = document.createDocumentFragment();
const items = ['苹果', '香蕉', '橙子'];

items.forEach(text => {
  const li = document.createElement('li');
  li.textContent = text;
  fragment.appendChild(li);  // 先放到 fragment，不触发页面重绘
});

document.querySelector('ul').appendChild(fragment);  // 一次性插入

遍历 DOM 节点

JavaScript
const child = document.querySelector('.item');

// 上下级关系
child.parentElement;           // 父元素
child.closest('nav');        // 向上找最近的 nav 祖先
child.children;                // 子元素集合（仅元素节点）
child.firstElementChild;
child.lastElementChild;

// 同级关系
child.nextElementSibling;      // 下一个兄弟元素
child.previousElementSibling;  // 上一个兄弟元素

// 遍历所有子元素
child.querySelectorAll('span');

DOM 就绪：DOMContentLoaded

JS 如果在 HTML 元素之前执行，会找不到元素。应等 DOM 解析完成后再操作：

JavaScript
// 方式一：script 放在 </body> 前（最简单）

// 方式二：监听 DOMContentLoaded
document.addEventListener('DOMContentLoaded', () => {
  const btn = document.querySelector('#myBtn');
  btn.addEventListener('click', () => {
    console.log('DOM 已就绪，可以安全操作元素');
  });
});

    10. 事件处理

    事件是用户或浏览器触发的动作：点击、输入、滚动、键盘按键、页面加载等。

    添加事件监听

    JavaScript
    const btn = document.querySelector('#myBtn');

// 推荐：addEventListener
btn.addEventListener('click', function(event) {
  console.log('按钮被点击了！');
  console.log(event.target);  // 触发事件的元素
});

// 箭头函数写法
btn.addEventListener('click', () => {
  alert('Hello!');
});

// 移除监听器
function handleClick() { ... }
btn.addEventListener('click', handleClick);
btn.removeEventListener('click', handleClick);

    常用事件

    事件触发时机
    click鼠标点击
    dblclick双击
    mouseenter / mouseleave鼠标进入/离开元素
    keydown / keyup键盘按下/抬起
    input输入框内容变化
    change输入框失去焦点且值改变
    submit表单提交
    scroll元素滚动
    load页面或资源加载完成
    DOMContentLoadedHTML 解析完成（不等图片）

    事件对象常用属性和方法

    JavaScript
    element.addEventListener('click', (e) => {
  e.preventDefault();    // 阻止默认行为（如链接跳转、表单提交）
  e.stopPropagation(); // 阻止事件冒泡
  e.target;              // 实际被点击的元素
  e.currentTarget;       // 绑定监听器的元素
});

// 表单提交示例
form.addEventListener('submit', (e) => {
  e.preventDefault();  // 阻止页面刷新
  const data = new FormData(form);
  console.log(data.get('username'));
});

    11. 异步入门

    JavaScript 是单线程的，但遇到耗时操作（网络请求、定时器）不会阻塞页面，靠的是异步机制。

    JavaScript
    // 延迟执行（毫秒）
const timerId = setTimeout(() => {
  console.log('3 秒后执行');

clearTimeout(timerId);  // 取消

// 重复执行
const intervalId = setInterval(() => {
  console.log('每秒执行');
}, 1000);
clearInterval(intervalId);  // 停止

    Promise

    JavaScript
    const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    resolve('成功的结果');
    // reject(new Error('失败原因'));
  }, 1000);
});
