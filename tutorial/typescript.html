TypeScript 基础教程

TypeScript（简称 TS）是 JavaScript 的超集——在 JS 基础上增加了静态类型系统。写代码时就能发现错误，大型项目更易维护。本教程假设你已学完 JavaScript 基础。

1. 什么是 TypeScript？

TypeScript 由微软开发，2012 年发布，目前已是前端工程化的事实标准。React、Vue、Angular 等主流框架都原生支持 TS。

核心概念：JS + 类型

所有合法的 JavaScript 代码，都是合法的 TypeScript 代码。TS 额外允许你声明「这个变量是什么类型」：

TypeScript
// JavaScript 写法 — TS 完全支持
let name = '小明';

// TypeScript 额外支持：显式标注类型
let age: number = 18;
let isStudent: boolean = true;

为什么需要 TypeScript？

  • 提前发现错误：类型不对在写代码时就会报错，不用等到运行时
  • 更好的编辑器体验：自动补全、跳转定义、重构更安全
  • 代码即文档：类型声明说明了函数期望什么参数、返回什么
  • 大型项目必备：多人协作、代码量大时，类型系统是安全网

重要：TS 最终会变成 JS

浏览器和 Node.js 不能直接运行 TypeScript。你需要用编译器（tsc）把 .ts 文件编译（转译）成 .js 后再运行。类型信息在编译后会被完全擦除，不影响运行时性能。

💡 学习前提 请先掌握 JavaScript 基础（变量、函数、对象、数组、ES6+ 语法）。TypeScript 不是一门新语言，而是给 JS 加了类型层。

2. TypeScript 与 JavaScript 的区别

这是新手最关心的问题。下面从多个维度对比，帮你建立清晰认知。

一句话总结

JavaScript 是运行在浏览器/Node 上的动态类型脚本语言。
TypeScript 是 JavaScript 的超集，增加了静态类型检查，编译后变成 JavaScript。

核心区别对照表

对比项 JavaScript TypeScript
类型检查 运行时才知道类型错误 编写代码时（编译期）就检查
类型声明 不需要，变量类型随时可变 可选但推荐显式声明类型
文件扩展名 .js / .mjs .ts / .tsx（含 JSX）
运行方式 浏览器/Node 直接运行 必须先编译成 JS 再运行
错误发现时机 运行时（用户可能遇到 bug） 开发时（编辑器 + 编译器报错）
IDE 支持 基础补全 精准补全、类型提示、安全重构
学习曲线 较低，入门快 需先会 JS，再学类型概念
适用场景 小脚本、快速原型、简单页面 中大型项目、团队协作、框架开发

同一段代码的对比

下面这个函数，在 JS 中不会报错，但运行结果可能是错的：

JavaScript
function add(a, b) {
  return a + b;
}

add(1, 2);       // 3 ✅
add('1', '2');     // '12' ⚠️ 字符串拼接，可能不是预期
add(1, '2');       // '12' ⚠️
add(1);            // NaN ⚠️ 少传参数，运行时才暴露
TypeScript
function add(a: number, b: number): number {
  return a + b;
}

add(1, 2);       // 3 ✅
add('1', '2');     // ❌ 编译报错：参数类型不匹配
add(1);            // ❌ 编译报错：缺少参数 b

JS 有但 TS 更严格的点

场景JavaScriptTypeScript
访问不存在的属性 返回 undefined，不报错 编译报错：Property 'xxx' does not exist
null / undefined 随意使用，运行时报错 默认不允许（可配置 strictNullChecks）
函数参数数量 多传少传都不报错 参数个数、类型必须匹配
对象结构 任意添加属性 必须符合 interface / type 定义
this 指向 运行时才知道 可在编译期标注 this 类型

TS 有但 JS 没有的特性

  • 类型注解：let x: number = 1
  • 接口（interface）：描述对象结构
  • 类型别名（type）：自定义类型名
  • 泛型（generics）：Array<string>、Promise<User>
  • 枚举（enum）：命名常量集合
  • 访问修饰符：public、private、protected
  • 工具类型：Partial<T>、Pick<T, K> 等
  • 联合类型：string | number

兼容性：JS 代码可以直接变成 TS 吗？

可以。 TypeScript 的设计目标就是平滑迁移：

  1. 把 .js 文件改名为 .ts
  2. 逐步添加类型，不必一次改完
  3. 设置 "allowJs": true 可让 JS 和 TS 文件共存

什么时候用 JS，什么时候用 TS？

选 JavaScript选 TypeScript
学习编程的前几周 已掌握 JS 基础，做正式项目
几十行的小工具脚本 React / Vue / Node 后端项目
快速验证想法的原型 多人协作、长期维护的代码库
不想配置编译流程 需要可靠的重构和自动补全
⚠️ 常见误解
  • TS 不是替代 JS，而是编译成 JS
  • TS 不会提升运行时性能（类型在编译后被擦除）
  • 学 TS 之前必须先会 JS，否则两头都不扎实

3. 环境搭建

安装 Node.js 和 TypeScript

终端
# 确认 Node.js 已安装（需 v18+ 推荐）
node -v
npm -v

# 全局安装 TypeScript 编译器
npm install -g typescript

# 查看版本
tsc -v

第一个 TypeScript 程序

hello.ts
const greeting: string = 'Hello, TypeScript!';
console.log(greeting);

function add(a: number, b: number): number {
  return a + b;
}

console.log(add(10, 20));
终端
# 编译：hello.ts → hello.js
tsc hello.ts

# 运行编译后的 JS
node hello.js

tsconfig.json 配置文件

实际项目中用配置文件统一管理编译选项：

tsconfig.json
{
  "compilerOptions": {
    "target": "ES2020",           // 编译目标 JS 版本
    "module": "ESNext",           // 模块系统
    "strict": true,              // 开启所有严格检查（推荐）
    "outDir": "./dist",           // 输出目录
    "rootDir": "./src",          // 源码目录
    "esModuleInterop": true,     // 兼容 CommonJS 模块
    "skipLibCheck": true         // 跳过库文件类型检查
  },
  "include": ["src/**/*"],       // 包含的文件
  "exclude": ["node_modules"]   // 排除的文件
}
终端
# 初始化 tsconfig.json
tsc --init

# 按 tsconfig 编译整个项目
tsc

# 监听模式：文件改动自动重新编译
tsc --watch

4. 基础类型

TypeScript 内置了与 JavaScript 对应的基础类型，并增加了一些额外类型。

TS 类型对应 JS 值示例
string字符串'hello', `模板`
number数字（含小数）42, 3.14, NaN
boolean布尔值true, false
null空值null
undefined未定义undefined
bigint大整数100n
symbol唯一标识Symbol('id')
any任意类型（跳过检查）尽量避免
unknown未知类型（安全的 any）需先检查再用
never永不存在的值抛出异常、无限循环
void无返回值函数不 return 时
TypeScript
let name: string = '张三';
let age: number = 25;
let active: boolean = true;

// 数组
let nums: number[] = [1, 2, 3];
let names: Array<string> = ['a', 'b'];  // 泛型写法，等价

// 元组 tuple：固定长度和类型的数组
let point: [number, number] = [10, 20];
let userInfo: [string, number, boolean] = ['小明', 18, true];

// any — 关闭类型检查，尽量不用
let anything: any = 1;
anything = 'hello';  // 不报错，但失去 TS 意义

// unknown — 更安全的 any，使用前必须收窄类型
let input: unknown = getUserInput();
if (typeof input === 'string') {
  console.log(input.toUpperCase());  // ✅ 此时 TS 知道 input 是 string
}

5. 类型注解与类型推断

显式类型注解

手动告诉 TS 变量/函数的类型：

TypeScript
let count: number = 0;
const title: string = '页面标题';

类型推断（Type Inference）

TS 会根据初始值自动推断类型，很多情况下不必手写：

TypeScript
let message = 'Hello';   // TS 推断为 string
let count = 42;           // TS 推断为 number
let isReady = true;       // TS 推断为 boolean

// message = 123;  // ❌ 报错：不能将 number 赋给 string

何时需要显式注解？

  • 声明变量时没有初始值：let result: number;
  • 函数参数和返回值（推断不够精确时）
  • 函数返回值类型复杂，想作为文档说明
  • 对象字面量需要约束结构时
💡 最佳实践 能推断就不手写。只在推断不准确或需要约束的地方加注解，保持代码简洁。

6. 接口（Interface）

Interface 用来描述对象的结构——有哪些属性、什么类型。这是 TS 最常用的特性之一。

TypeScript
interface User {
  id: number;
  name: string;
  email: string;
  age?: number;          // ? 表示可选属性
  readonly createdAt: Date; // readonly 不可修改
}

const user: User = {
  id: 1,
  name: '张三',
  email: 'zhangsan@example.com',
  createdAt: new Date(),
};

// user.createdAt = new Date();  // ❌ 只读属性不能改
// user.phone = '123';           // ❌ 接口未定义 phone 属性

接口描述函数

TypeScript
interface SearchFunc {
  (keyword: string, page: number): string[];
}

const search: SearchFunc = (keyword, page) => {
  return [`结果: ${keyword} 第${page}页`];
};

接口继承与扩展

TypeScript
interface Animal {
  name: string;
}

interface Dog extends Animal {
  breed: string;
  bark(): void;
}

const myDog: Dog = {
  name: '旺财',
  breed: '柴犬',
  bark() { console.log('汪汪!'); },
};

7. 联合类型与类型别名

联合类型（Union Types）

值可以是多种类型之一，用 | 连接：

TypeScript
let id: string | number;
id = 'abc123';  // ✅
id = 123;       // ✅
// id = true;   // ❌

function printId(id: string | number) {
  if (typeof id === 'string') {
    console.log(id.toUpperCase());
  } else {
    console.log(id.toFixed(2));
  }
}

类型别名（Type Alias）

用 type 给类型起名字，可描述联合类型、对象、函数等：

TypeScript
type ID = string | number;
type Status = 'pending' | 'success' | 'error';  // 字面量联合

type Point = {
  x: number;
  y: number;
};

type Callback = (data: string) => void;

interface vs type 怎么选？

场景推荐
描述对象结构interface（可扩展、可继承）
联合类型、交叉类型type
字面量类型type
需要声明合并（同名接口自动合并）interface
函数类型两者均可，type 更简洁

8. 函数类型

TypeScript
// 参数类型 + 返回值类型
function greet(name: string): string {
  return `你好，${name}！`;
}

// 箭头函数
const multiply = (a: number, b: number): number => a * b;

// 可选参数
function buildName(first: string, last?: string): string {
  return last ? `${first} ${last}` : first;
}

// 默认参数（自动推断类型）
function createUser(name: string, role = 'user'): void {
  console.log(name, role);
}

// 剩余参数
function sum(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}

// 函数类型作为参数
function doTask(callback: (result: string) => void) {
  callback('完成');
}

函数重载（Overloads）

同一函数根据不同参数类型返回不同结果：

TypeScript
function format(value: string): string;
function format(value: number): string;
function format(value: string | number): string {
  return typeof value === 'string' ? value.trim() : value.toFixed(2);
}

format(' hello ');  // 'hello'
format(3.1415);     // '3.14'

9. 类（Class）

TS 的类比 JS 多了访问修饰符和更严格的类型检查。

TypeScript
class Animal {
  // 访问修饰符
  public name: string;       // 公开（默认）
  private age: number;        // 仅类内部访问
  protected species: string; // 类及子类可访问
  readonly id: number;       // 只读

  constructor(name: string, age: number, species: string) {
    this.name = name;
    this.age = age;
    this.species = species;
    this.id = Math.random();
  }

  speak(): void {
    console.log(`${this.name} 发出声音`);
  }
}

class Dog extends Animal {
  constructor(name: string, age: number, public breed: string) {
    super(name, age, '犬科');  // 调用父类构造函数
  }

  speak(): void {  // 方法重写
    console.log(`${this.name} 说：汪汪！`);
  }
}

const dog = new Dog('旺财', 3, '柴犬');
dog.speak();
// dog.age;  // ❌ private 属性外部不可访问

实现接口

TypeScript
interface Printable {
  print(): void;
}

class Document implements Printable {
  print(): void {
    console.log('打印文档...');
  }
}

10. 泛型（Generics）

泛型让类型也能「参数化」，写一次代码适配多种类型，是 TS 最强大的特性之一。

TypeScript
// 没有泛型：返回值是 any，丢失类型信息
function getFirst(arr: any[]): any {
  return arr[0];
}

// 有泛型：保留类型信息
function getFirst<T>(arr: T[]): T | undefined {
  return arr[0];
}

const num = getFirst([1, 2, 3]);     // number | undefined
const str = getFirst(['a', 'b']);  // string | undefined

// 泛型接口
interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
}

interface User {
  id: number;
  name: string;
}

const res: ApiResponse<User> = {
  code: 200,
  message: 'ok',
  data: { id: 1, name: '张三' },
};

// 泛型约束：T 必须有 length 属性
function logLength<T extends { length: number }>(item: T): void {
  console.log(item.length);
}

logLength('hello');     // ✅ string 有 length
logLength([1, 2, 3]);  // ✅ array 有 length
// logLength(123);  // ❌ number 没有 length

常见泛型使用场景

  • Array<T> / T[] — 数组
  • Promise<T> — 异步返回值
  • Map<K, V> / Set<T> — 集合
  • Record<K, V> — 键值对对象

11. 枚举与字面量类型

枚举（Enum）

TypeScript
// 数字枚举
enum Direction {
  Up,     // 0
  Down,   // 1
  Left,   // 2
  Right,  // 3
}

let dir: Direction = Direction.Up;

// 字符串枚举（更常用，可读性好）
enum HttpStatus {
  OK = '200',
  NotFound = '404',
  ServerError = '500',
}

字面量类型

TypeScript
type Theme = 'light' | 'dark';
type Size = 'sm' | 'md' | 'lg';

function setTheme(theme: Theme) {
  document.body.dataset.theme = theme;
}

setTheme('dark');   // ✅
// setTheme('blue');  // ❌ 只能是 'light' 或 'dark'

// as const 让值变成字面量类型
const config = {
  api: 'https://api.example.com',
  timeout: 5000,
} as const;
// config.api 的类型是 'https://api.example.com'，不是 string
💡 现代 TS 趋势 很多团队更倾向用 type Status = 'pending' | 'done' 代替 enum，因为 enum 会生成额外 JS 代码，而字面量联合类型编译后会被擦除。

12. 类型收窄（Type Narrowing）

当变量是联合类型时，需要通过检查「收窄」到具体类型，TS 才能允许你使用该类型的方法。

TypeScript
function padLeft(value: string, padding: string | number) {
  // typeof 类型守卫
  if (typeof padding === 'number') {
    return ' '.repeat(padding) + value;
  }
  return padding + value;
}

// instanceof 检查
function logDate(input: Date | string) {
  if (input instanceof Date) {
    console.log(input.toISOString());
  } else {
    console.log(input.toUpperCase());
  }
}

// in 操作符
type Cat = { meow(): void };
type Dog = { bark(): void };

function speak(animal: Cat | Dog) {
  if ('meow' in animal) {
    animal.meow();
  } else {
    animal.bark();
  }
}

// 自定义类型守卫
function isString(value: unknown): value is string {
  return typeof value === 'string';
}

14. 常用工具类型

TypeScript 内置了许多工具类型，用于变换已有类型：

TypeScript
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

// Partial<T> — 所有属性变可选
type PartialUser = Partial<User>;
// { id?: number; name?: string; email?: string; age?: number }

// Required<T> — 所有属性变必填
type RequiredUser = Required<PartialUser>;

// Pick<T, K> — 选取部分属性
type UserBrief = Pick<User, 'id' | 'name'>;
// { id: number; name: string }

// Omit<T, K> — 排除部分属性
type UserWithoutEmail = Omit<User, 'email'>;

// Record<K, V> — 构造键值对类型
type RolePermissions = Record<'admin' | 'user', string[]>;

// Readonly<T> — 所有属性变只读
type ReadonlyUser = Readonly<User>;

15. 最佳实践

✅ 推荐做法

  • 开启 "strict": true
  • 优先用 interface 描述对象，用 type 描述联合/交叉类型
  • 避免 any，用 unknown 代替需要后续收窄的场景
  • 让 TS 推断简单类型，只在必要处手写注解
  • 用字面量联合类型代替 magic string
  • 给公共 API（导出函数、组件 props）写明确类型
  • 使用 ESLint + TypeScript 插件保持代码质量

❌ 避免的做法

  • 到处用 as any 绕过类型检查
  • 过度注解（每个变量都写类型，代码冗余）
  • 不开启 strict 模式（会错过 TS 的核心价值）
  • 没学 JS 就直接学 TS（基础不牢）

学习路线建议

  1. 掌握 JavaScript ES6+ 基础
  2. 理解 TS 基础类型和 interface
  3. 在现有 JS 项目中尝试把文件改为 .ts
  4. 学习泛型和工具类型
  5. 用 TS 做一个 React 或 Node 小项目
