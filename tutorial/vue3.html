Vue 3 基础教程

Vue 3 是目前最流行的前端框架之一，帮你用组件化 + 响应式的方式构建用户界面。本教程假设你已掌握 HTML、CSS、JavaScript 基础，TypeScript 知识会有帮助但不是必须。

1. 什么是 Vue 3？

Vue（读音 /vjuː/，像 view）是一个用于构建用户界面的渐进式 JavaScript 框架。Vue 3 是 2020 年发布的最新 major 版本，性能更好，并引入了组合式 API（Composition API）。

Vue 能帮你做什么？

  • 把页面拆成可复用的组件（按钮、导航栏、卡片……）
  • 数据变化时自动更新界面，不用手动操作 DOM
  • 管理复杂应用的状态（用户信息、购物车、表单……）
  • 配合 Vue Router、Pinia 构建单页应用（SPA）

「渐进式」是什么意思？

你可以只在一个页面里用 Vue 处理一小块交互，也可以用它构建整个大型应用——按需采用，没有「全有或全无」的压力。

💡 学习前提 请先掌握 JavaScript（尤其是 ES6+、DOM、模块化）。有 TypeScript 基础会更容易上手 Vue 3 + TS 项目。

2. Vue 与原生 JavaScript 的区别

对比项原生 JSVue 3
更新界面 手动 querySelector + 改 textContent / classList 改数据，Vue 自动更新 DOM
代码组织 HTML / CSS / JS 分离，大项目难维护 组件化：每个组件包含模板 + 逻辑 + 样式
状态管理 自己维护变量，手动同步到 DOM 响应式系统，数据和视图自动同步
复用 复制粘贴或自己封装函数 组件直接复用，传 props 即可
学习曲线 低（基础部分） 需先会 JS，再学框架概念

同功能对比：计数器

原生 JS — 你要自己找元素、改文字：

JavaScript
let count = 0;
const display = document.querySelector('#count');
const btn = document.querySelector('#btn');

btn.addEventListener('click', () => {
  count++;
  display.textContent = count;  // 手动同步 DOM
});

Vue 3 — 只改数据，界面自动更新：

Vue 3
const { createApp, ref } = Vue;

createApp({
  setup() {
    const count = ref(0);
    const increment = () => count.value++;
    return { count, increment };
  }
}).mount('#app');

<!-- 模板 -->
<div id="app">
  <p>{{ count }}</p>
  <button @click="increment">+1</button>
</div>

3. 环境搭建

方式一：CDN 快速体验（学习用）

无需安装，在 HTML 里引入 Vue 即可，适合理解概念：

HTML
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

方式二：Vite 创建项目（正式开发推荐）

终端
# 需要 Node.js 18+
npm create vue@latest my-vue-app

# 按提示选择：TypeScript? Router? Pinia? 等
cd my-vue-app
npm install
npm run dev     # 启动开发服务器，通常 http://localhost:5173

项目结构（Vite + Vue 3）

目录
my-vue-app/
├── public/              # 静态资源
├── src/
│   ├── assets/          # 图片、样式等
│   ├── components/      # 可复用组件
│   ├── views/           # 页面级组件（配合 Router）
│   ├── App.vue          # 根组件
│   └── main.js          # 入口文件
├── index.html           # HTML 入口
├── package.json
└── vite.config.js       # Vite 配置

单文件组件（.vue）

Vue 项目的核心单元，一个文件里包含模板、逻辑、样式：

HelloWorld.vue
<!-- 模板：HTML 结构 -->
<template>
  <div class="hello">
    <h1>{{ msg }}</h1>
    <button @click="count++">点击 {{ count }} 次</button>
  </div>
</template>

<!-- 逻辑：JavaScript -->
<script setup>
import { ref } from 'vue'
const msg = ref('Hello Vue 3!')
const count = ref(0)
</script>

<!-- 样式：CSS（只作用于本组件） -->
<style scoped>
.hello { text-align: center; padding: 2rem; }
h1 { color: #42b883; }
</style>

4. 第一个 Vue 应用

HTML + Vue 3 CDN
<!doctype html>
<html lang="zh-CN">
<head>
  <meta charset="utf-8">
  <title>我的第一个 Vue 应用</title>
</head>
<body>

  <div id="app">
    <h1>{{ message }}</h1>
    <p>计数：{{ count }}</p>
    <button @click="count++">+1</button>
  </div>

  <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
  <script>
    const { createApp, ref } = Vue

    createApp({
      setup() {
        const message = ref('Hello Vue 3!')
        const count = ref(0)
        return { message, count }
      }
    }).mount('#app')
  </script>
</body>
</html>

三个关键步骤：

  1. createApp({ ... }) — 创建 Vue 应用实例
  2. setup() — 定义响应式数据和方法，return 出去供模板使用
  3. .mount('#app') — 挂载到 HTML 中 id 为 app 的元素上

6. 响应式数据

Vue 3 的核心：数据变了，界面自动更新。组合式 API 中主要用 ref 和 reactive。

ref — 包装单个值

JavaScript
import { ref } from 'vue'

const count = ref(0)       // 数字
const name = ref('小明')   // 字符串
const flag = ref(true)     // 布尔

// JS 中读写要用 .value
count.value++            // ✅
console.log(count.value) // ✅

// 模板中自动解包，不需要 .value
// {{ count }} 即可

reactive — 包装对象

JavaScript
import { reactive } from 'vue'

const user = reactive({
  name: '张三',
  age: 25,
})

user.age++               // 直接改属性，不需要 .value
user.city = '北京'       // 新增属性也是响应式的

ref vs reactive 怎么选？

场景推荐
基本类型（string、number、boolean）ref
需要整体替换的对象ref
复杂对象、表单数据reactive
不确定时优先 ref（更通用）
Vue 3 实时演示 — 响应式计数器（本页加载 Vue CDN）

{{ count }}

改 count 变量，界面自动更新，无需手动操作 DOM

7. 计算属性与侦听器

computed — 派生数据

基于现有数据计算出的值，有缓存，依赖不变不会重新计算：

JavaScript
import { ref, computed } from 'vue'

const firstName = ref('张')
const lastName = ref('三')

const fullName = computed(() => {
  return firstName.value + lastName.value
})

// 模板：{{ fullName }}  →  "张三"

watch — 监听变化

数据变化时执行副作用（请求 API、存 localStorage 等）：

JavaScript
import { ref, watch } from 'vue'

const keyword = ref('')

watch(keyword, (newVal, oldVal) => {
  console.log(`搜索词从 "${oldVal}" 变为 "${newVal}"`)
  // 可以在这里发 API 请求
})

8. 条件与列表渲染

Vue Template
<!-- 条件渲染 -->
<p v-if="score >= 60">及格了！</p>
<p v-else-if="score >= 50">再努力一下</p>
<p v-else>不及格</p>

<!-- v-show：只切换 display，适合频繁切换 -->
<div v-show="visible">内容</div>

<!-- 列表渲染 — 必须加 :key -->
<ul>
  <li v-for="item in todos" :key="item.id">
    {{ item.text }}
  </li>
</ul>

<!-- 遍历对象 -->
<li v-for="(value, key) in user" :key="key">
  {{ key }}: {{ value }}
</li>
⚠️ v-if vs v-show v-if 真正创建/销毁 DOM，适合不常切换；v-show 只改 CSS display，适合频繁显隐（如 Tab 切换）。

9. 事件处理与 v-model

Vue Template
<!-- 事件 -->
<button @click="handleClick">点击</button>
<button @click="count++">内联</button>
<button @click="say('hello')">传参</button>
<form @submit.prevent="onSubmit">...</form>  <!-- .prevent 阻止默认行为 -->

<!-- v-model 双向绑定 -->
<input v-model="username" placeholder="用户名">
<textarea v-model="bio"></textarea>
<input type="checkbox" v-model="agreed">
<select v-model="city">
  <option value="bj">北京</option>
  <option value="sh">上海</option>
</select>

<p>你好，{{ username }}！</p>  <!-- 输入框一改，这里自动变 -->

v-model 本质上是 :value + @input 的语法糖，让你不用手动写 DOM 同步逻辑。

10. 组件基础

组件是 Vue 的核心思想：把 UI 拆成独立、可复用的块。每个组件有自己的模板、逻辑和样式。

MyButton.vue
<template>
  <button class="my-btn" @click="handleClick">
    {{ label }}
  </button>
</template>

<script setup>
const label = '点击我'
function handleClick() {
  alert('按钮被点击了！')
}
</script>

<style scoped>
.my-btn {
  background: #42b883;
  color: white;
  border: none;
  padding: 0.5rem 1rem;
  border-radius: 6px;
  cursor: pointer;
}
</style>
App.vue 中使用
<template>
  <MyButton />
  <MyButton />  <!-- 复用 -->
</template>

<script setup>
import MyButton from './components/MyButton.vue'
</script>

11. Props 与 Emits

组件之间通信的两个核心机制：父传子用 props，子传父用 emits。

Props — 父组件向子组件传数据

UserCard.vue
<template>
  <div class="card">
    <h3>{{ name }}</h3>
    <p>{{ age }} 岁</p>
  </div>
</template>

<script setup>
// 声明接收哪些 props
const props = defineProps({
  name: String,
  age: Number,
})
</script>

<!-- 父组件 -->
<UserCard name="张三" :age="25" />

Emits — 子组件向父组件发事件

LikeButton.vue
<template>
  <button @click="emit('like', id)">点赞</button>
</template>

<script setup>
defineProps({ id: Number })
const emit = defineEmits(['like'])

// 父组件监听：
// <LikeButton :id="1" @like="handleLike" />
</script>

12. 组合式 API（Composition API）

Vue 3 推荐的代码组织方式。用 <script setup> 语法，逻辑按功能组织，而不是按选项（data、methods……）分散。

Options API vs Composition API

Options API（Vue 2 风格）Composition API（Vue 3 推荐）
写法data()、methods、computed 分开setup() 或 <script setup>
逻辑复用Mixins（容易冲突）Composables 函数（清晰）
TypeScript支持一般原生友好
推荐度老项目维护新项目首选

Composable — 逻辑复用函数

composables/useCounter.js
import { ref } from 'vue'

// 可复用的计数器逻辑
export function useCounter(initial = 0) {
  const count = ref(initial)
  const increment = () => count.value++
  const decrement = () => count.value--
  const reset = () => { count.value = initial }
  return { count, increment, decrement, reset }
}
组件中使用
<script setup>
import { useCounter } from '@/composables/useCounter'
const { count, increment, decrement, reset } = useCounter(10)
</script>

<template>
  <p>{{ count }}</p>
  <button @click="increment">+</button>
</template>

13. 生命周期

组件从创建到销毁会经历一系列阶段，可以在特定时机执行代码：

钩子时机常见用途
onMounted组件挂载到 DOM 后请求 API、初始化第三方库、聚焦输入框
onUpdated数据变化导致 DOM 更新后较少用，注意性能
onUnmounted组件销毁前清理定时器、取消订阅
JavaScript
import { ref, onMounted, onUnmounted } from 'vue'

const data = ref([])

onMounted(async () => {
  const res = await fetch('/api/users')
  data.value = await res.json()
})

let timer
onMounted(() => {
  timer = setInterval(() => console.log('tick'), 1000)
})
onUnmounted(() => {
  clearInterval(timer)  // 组件销毁时清理，防止内存泄漏
})

15. 最佳实践

✅ 推荐

  • 新项目用 <script setup> + 组合式 API
  • 组件名用 PascalCase（UserCard.vue）
  • v-for 始终加 :key
  • 复杂逻辑抽到 composable 函数
  • 样式用 scoped 避免污染全局
  • 安装 Vue DevTools 浏览器插件

❌ 避免

  • 没学 JS 就直接学 Vue
  • 在模板里写复杂逻辑（用 computed 代替）
  • 直接修改 props（应 emit 事件让父组件改）
  • 滥用 v-html（XSS 风险）

学习路线

  1. HTML / CSS / JavaScript 基础
  2. 用 CDN 写几个小 demo（计数器、待办列表）
  3. npm create vue@latest 创建项目
  4. 学组件、props、emits、v-model
  5. 学 Vue Router + Pinia
  6. 做一个完整小项目（博客、商城、管理后台）
